## 3. Найденные Дефекты До Исправления

### P1. Устаревшая Точка Входа

`README.md`, `AGENTS.md` и карта документации называли аудит `78` текущим. После него появились документы `79-81`, изменились шаблоны товарной карточки и контактов, а также внутренние ссылки контента. Поэтому `78` уже не мог подтверждать полную синхронизацию текущего состояния.

Исправление: создан аудит `82`; `78` помечен историческим снимком.

### P1. Неполная Русификация Новых Документов

В документах `79-81` и товарном шаблоне `10` оставались английские названия разделов, метрик и редакционных рекомендаций: `About`, `Contact`, `Definition Of Done`, `Body links`, `Orphan pages`, `Product Image Layer`, `shot list` и другие.

Исправление: пользовательские пояснения, заголовки и метрики переведены на русский. Технические идентификаторы и официальные названия сохранены без искажения.

### P1. Неточная Формулировка Об Индексации

`README.md` создавал впечатление, что любая production-сборка возвращает `noindex`. Фактическое поведение различается:

- локальная команда `npm run build:production` использует environment `production` и выводит `index,follow` для индексируемых страниц;
- опубликованный Netlify-сайт пока собирается общей командой с `--environment development` и поэтому закрыт `noindex,nofollow`.

Исправление: формулировка привязана к текущей Netlify-конфигурации, а не к production-сборке вообще.

### P2. Несогласованные Подписи Дат

Часть действующих регламентов использовала `Актуально на`, а аудит `80` имел нестандартную подпись даты.

Исправление:

- действующие регламенты используют `Обновлено: YYYY-MM-DD.`;
- аудиты используют `Дата аудита: YYYY-MM-DD.`;
- документ `80` отдельно фиксирует дату исходного аудита и дату обновления после перелинковки.

## 6. Проверка Hugo И Tailwind CSS

Официальная документация Hugo для `css.TailwindCSS` подтверждает текущую архитектуру проекта:

- требуется Tailwind CSS версии 4 или выше;
- Hugo вызывает Tailwind CSS CLI из npm-зависимостей проекта;
- standalone Tailwind CSS CLI не поддерживается начиная с Hugo `0.161.0`;
- `tailwindcss` и `@tailwindcss/cli` должны оставаться npm-зависимостями.

Следовательно, документация проекта по Hugo `0.163.0` и Tailwind CSS `4.3.0` актуальна. Удалять npm-зависимости или переходить на standalone CLI нельзя.

Текущий контракт `81` соответствует Google Search Central:

## 10. Оставшиеся Риски Проекта

Эти пункты важны, но не являются ошибками составления документации:

1. **P0: основной домен закрыт от индексации текущей Netlify-командой.** Перед production-релизом нужен отдельный production context с `hugo --environment production`.
2. **P1: товарные изображения недостоверны.** Аудит `80` фиксирует одинаковые главные изображения и тестовые SKY-файлы. Нужны официальные изображения конкретных SKU.
3. **P1: нет постоянной команды аудита ссылок.** Проверка документа `81` пока остается отдельной процедурой.
4. **P2: нет 28-дневных Search Console / CrUX данных.** Netlify RUM уже подтверждает хорошие реальные CWV за 24 часа, но этого недостаточно для вывода по Google field data, SEO, CTR и поисковым позициям.
5. **P2: доступны патч-обновления Hugo и Tailwind CSS.** Обновлять только после отдельной проверки совместимости.

## 12. Правило Дальнейшей Поддержки

После любого изменения кода или контента нужно обновлять профильный постоянный документ, а не только добавлять новый аудит.

Новый аудит нужен, когда изменяется сразу несколько подсистем или требуется зафиксировать подтвержденное состояние на конкретную дату. Исторический аудит нельзя продолжать называть текущим после появления более нового полного аудита.
