## 6. Сверка С Лучшими Практиками 2026

Текущая документация согласована с ключевыми практиками:

- Google Search Central требует доступные для обхода ссылки, понятные анкоры, корректные canonical/hreflang/sitemap и полезный контент для людей; это закрывают документы `27`, `28`, `76`, `81`, `84`, `85`, `87`, `88`.

- Google Article structured data рекомендует репрезентативные изображения и набор соотношений **16:9**, **4:3**, **1:1**; это зафиксировано в документах `05`, `06`, `67`, `79`, `89`.
- Core Web Vitals сохраняют проектные ориентиры `LCP`, `INP`, `CLS`; это закрывают документы `12`, `13`, `14`, `83`.
- Для AI Search документация усиливает видимый текст, сущности, внутренние ссылки и structured data, а не обещает “магическую” схему; это закрывают документы `24`, `25`, `26`, `79`, `85`, `88`.

Активные документы соответствуют стандарту:

- есть H1 и дата `Обновлено:` или `Дата аудита:`;

- placeholder-ссылки в документах-шаблонах `08-11` являются полями для заполнения, а не реальными ссылками проекта.
