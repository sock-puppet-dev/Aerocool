## Принципы SEO 2026

Стратегия строится на официальных принципах Google Search Central и на текущей архитектуре проекта:

- SEO помогает поисковым системам понимать контент, а пользователям — находить сайт и решать, стоит ли на него переходить.
- Нет набора секретных действий, который гарантирует первое место в Google.
- Страницы должны быть полезными, надежными и написанными для людей, а не ради механического повторения запросов.
- Сайт должен быть логически организован: отдельные URL для разных интентов, понятные директории, внутренние ссылки и корректные sitemap/canonical/hreflang.
- Для e-commerce важны не только ключевые слова, но и подтвержденные товарные данные, видимые условия покупки, structured data и доверительный слой.
- Search Console должна стать главным источником фактической проверки спроса: показы, клики, CTR, средняя позиция, страна и устройство.

Практический вывод: развивать нужно не “больше ключей”, а “лучшие ответы под понятные поисковые задачи”.

## Архитектура Семантического Ядра

| Уровень | Роль | Primary URL |
| --- | --- | --- |
| Бренд | Навигация, доверие, официальный источник | `/`, `/about/`, `/products/` |
| Категория | Широкий коммерческий спрос | `/products/`, `/articles/how-to-choose-aerocool-chair/` |
| Серия | Выбор линейки | `/products/sky/`, `/products/wing/`, `/products/xtal/` |
| Товар | Модель, цвет, SKU-интент, purchase | конкретные страницы товаров |
| Сравнение | Выбор между моделями, сериями, материалами | статьи-сравнения |
| Сценарий | Работа, home office, гейминг, долгий день, команда | evergreen-статьи |
| Support | Гарантия, доставка, контакты, права на изображения, уход | `/faq/`, `/contact/`, `/image-license/`, профильные статьи |
| News | Запуски, обновления каталога, поддержка бренда | `/news/` и отдельные новости |

Каждая URL должна иметь один главный интент. Вторичные ключи допустимы только как поддержка основного ответа.

## Приоритеты Развития

### P0: Защитить Деньги И Навигацию

Цель: не потерять брендовый, каталоговый и товарный спрос.

- Держать главную, каталог, серии и товары синхронными в `uk/ru`.
- Не размывать товары общими информационными фразами.
- Не создавать отдельные страницы под фильтры каталога без отдельного SEO-решения.
- Поддерживать внутренние ссылки из статей к сериям, товарам, FAQ и контактам.

### P1: Усилить Коммерческое Исследование

Цель: забрать пользователя на этапе выбора.

- Расширять страницы про выбор кресла, home office, работу за компьютером, долгие сессии и сравнение серий.
- Усиливать материаловый гайд блоками про уход, вентиляцию, контакт с поверхностью и сценарии помещения.
- Добавлять FAQ-блоки к существующим URL, если вопрос поддерживает главный интент страницы.
- Поддерживать B2B-сценарии в статье про офисную команду: переговорная, рабочие места, команды разработчиков.

### P2: Проверять Long-Tail Перед Новыми Страницами

Цель: не плодить слабые страницы.

- `уход за leatherette`, `уход за mesh`, `WING Mesh или WING Loft Air`, `XTAL Racer или XTAL Mesh` уже заведены в базу и должны сначала усиливать текущий материаловый гайд.
- `кресла для переговорной` и `кресла для команды разработчиков` сначала раскрывать внутри B2B-гайда.
- Отдельную страницу создавать только если GSC или рекламные кампании покажут отдельный устойчивый спрос и другой интент.

## План Расширения На 2026

### Шаг 1. Сразу После Этого Обновления

- Усилить существующие статьи блоками под новые CSV-кластеры.
- В материаловый гайд добавить практические подразделы про уход, вентиляцию и выбор поверхности внутри `WING/XTAL`.
- В B2B-гайд добавить подразделы про переговорную и команду разработчиков.
- В каталог и контакты добавить аккуратные purchase-мостики без обещаний цены или наличия.

### Шаг 2. После Подключения Google Search Console

- Заполнить `gsc_impressions`, `gsc_clicks`, `gsc_ctr`, `gsc_avg_position`, `country`, `device`, `last_checked`.
- Найти строки с показами и низким CTR: переписать `title`, `description`, вводный экран и внутренние ссылки.
- Найти строки с хорошими показами и плохой позицией: усилить глубину страницы и доказательные блоки.
- Разделить сильные строки `any` на отдельные `uk` и `ru`, если языковые запросы отличаются.

### Шаг 3. После Подтверждения Бизнес-Фактов

- Усилить `цена`, `наличие`, `доставка`, `гарантия`, `оплата`, `возврат` только подтвержденными данными.
- Синхронизировать товарный front matter, видимый текст, `/faq/` и schema.org `Offer`.
- Не добавлять фразы “в наличии”, “лучшая цена”, “официальная гарантия” без подтвержденного источника правды.

### Шаг 4. После Production-Проверки

- Проверить индексируемость ключевых URL через Search Console.
- Сравнить реальные страницы входа с `target_url` в CSV.
- Обновить приоритеты `P0/P1/P2/P3` по фактическим показам, кликам и бизнес-ценности.
- Создавать новые посадочные страницы только там, где текущая URL не закрывает интент.

## Рекомендации По Контенту

- Писать блоки как ответы на вопросы пользователя, а не как перечисление ключей.
- Держать `title`, `description`, `H1`, первый экран и внутренние ссылки в одном интенте.
- Для “лучшее/лучший/краще” всегда объяснять критерии выбора.
- Для технических терминов давать короткое объяснение рядом с термином.
- Для товаров связывать коммерческий текст с реальными характеристиками, а не с общими обещаниями.
- Для `uk` и `ru` сохранять синхронную структуру, но формулировать ключи естественно для каждого языка.

## Чего Не Делать

- Не использовать `meta keywords` как SEO-стратегию.
- Не создавать отдельные страницы под каждую перестановку слов.
- Не индексировать фильтры каталога без отдельного решения.
- Не смешивать на одной странице `купить`, `как выбрать`, `что это`, `сравнение` и `после покупки`, если главный интент теряется.
- Не добавлять неподтвержденные коммерческие обещания в тексты, FAQ или structured data.
- Не расширять семантику за счет медицинских обещаний про осанку.

## Метрики Успеха

| Метрика | Как Использовать |
| --- | --- |
| GSC impressions | Проверять реальный спрос по кластерам. |
| GSC clicks | Оценивать, какие URL уже получают трафик. |
| GSC CTR | Искать проблемы сниппета и несоответствие ожиданиям. |
| GSC average position | Определять страницы, которым нужна глубина, ссылки или уточнение интента. |
| Assisted inquiries | Связывать organic landing pages с заявками через `/contact/`. |
| Indexed pages | Проверять, что важные URL доступны Google. |
| Cannibalization cases | Ищется по одному запросу, который получает показы на нескольких слабых URL. |

## Итоговая Стратегия

Проект уже имеет сильное семантическое основание: бренд, категории, серии, товары, сравнения, материалы, механики, сценарии использования, новости и support-слой связаны между собой. Дальнейший рост должен идти через три действия:

1. Усиление существующих страниц под уже заведенные кластеры.
2. Подключение Search Console и пересчет приоритетов по реальным данным.
3. Создание новых посадочных страниц только там, где текущий URL не закрывает интент и есть подтвержденный спрос.

Главный принцип для 2026 года: каждая новая ключевая фраза должна либо улучшать существующую страницу, либо доказывать необходимость новой страницы данными.
