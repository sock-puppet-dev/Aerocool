## Главный Принцип

Новые ключевые слова не должны просто добавляться в текст. Каждый ключ должен получить один из трех статусов:

| Статус | Что Делать |
| --- | --- |
| Усилить существующую страницу | Добавить блок, FAQ, таблицу, сравнение, внутреннюю ссылку или уточнение сниппета. |
| Держать как growth-строку | Завести в CSV, но не создавать страницу до данных GSC или рекламного теста. |
| Создать новую страницу | Только если текущая URL не закрывает интент и есть подтвержденный спрос. |

## Как Грамотно Использовать Ключи

### 1. На Уровне Страницы

Для каждой страницы выбирать:

- один главный интент;
- 3-7 поддерживающих ключей;
- 2-4 внутренних ссылки на следующие шаги;
- 1-3 FAQ-вопроса, если они реально помогают пользователю;
- один понятный следующий шаг: каталог, серия, товар, FAQ или контакты.

Не нужно вставлять все похожие ключи в первый экран. Лучше распределять их по смысловым блокам.

### 2. В `title`, `description` И `H1`

`title` должен работать на сниппет и главный интент. `H1` должен звучать естественно для пользователя. Дополнительные ключи лучше выносить в `description`, `H2`, первые абзацы и FAQ.

Пример:

| Плохо | Лучше |
| --- | --- |
| `Кресло для программиста, IT, работы, офиса, компьютера` | `Как выбрать кресло для работы за компьютером: посадка, подлокотники и вентиляция` |

### 3. В Заголовках Разделов

`H2` и `H3` должны отвечать на реальные вопросы:

- `Когда Mesh лучше для теплой комнаты`;
- `Как выбрать кресло по росту и посадке`;
- `Что важно для видеозвонков в home office`;
- `Как не ошибиться с креслами для open space`;
- `Где проверять цену и наличие конкретной модели`.

### 4. В FAQ

FAQ использовать только там, где есть короткий практический ответ. Не превращать FAQ в скрытый список ключей.

Хороший FAQ:

| Вопрос | Целевая страница |
| --- | --- |
| `Подойдет ли Mesh для теплой комнаты?` | Материаловый гайд |
| `Как настроить поясничную поддержку?` | Гайд по регулировкам |
| `Где уточнить наличие модели?` | Товар + контакты |
| `Есть ли гарантийный ремонт?` | FAQ + контакты |

### 5. Во Внутренних Ссылках

Анкор должен объяснять следующий шаг, а не просто повторять ключ. Примеры:

- `сравните SKY Light и SKY 360`;
- `откройте гайд по материалам Racer, Loft Air и Mesh`;
- `проверьте цену и наличие на странице модели`;
- `перейдите в FAQ по гарантии и сервису`.

## Roadmap Расширения Контента

### Этап 1. Усилить Существующие Страницы

Сначала расширять уже опубликованные URL:

| URL | Что Добавить |
| --- | --- |
| `/articles/chair-for-computer-work/` | Блоки про программиста, IT-специалиста, рабочий день, экран, стол и перерывы. |
| `/articles/best-chair-for-home-office/` | Видеозвонки, небольшой кабинет, светлая комната, нейтральный внешний вид. |
| `/articles/racer-vs-loft-air-vs-mesh/` | Теплая комната, сетчатая спинка, уход, проветривание, выбор поверхности. |
| `/articles/how-to-choose-chair-by-adjustability/` | Рост, сиденье, подлокотники, поясничная поддержка, сценарии настройки. |
| `/articles/chairs-for-office-team/` | Open space, IT-команда, переговорная, единый стиль рабочих мест. |
| `/articles/chair-setup-after-purchase/` | Сборка, первая настройка, порядок проверки после покупки. |
| `/faq/` | Гарантийный ремонт, сервис, где уточнить наличие, как связаться. |
| `/contact/` | Где купить, где посмотреть, консультация, сервисный маршрут без обещания шоурума. |
| Товарные страницы | Цена, наличие, характеристики, SKU/MPN, `priceValidUntil`, сравнение с соседними моделями, FAQ по покупке. |

### Этап 2. Создавать Новые Страницы Только После Данных

Кандидаты на будущие страницы:

| Возможная Страница | Когда Создавать |
| --- | --- |
| `кресло для программиста` | Если GSC покажет отдельные показы/клики, которые не закрывает общий гайд по работе за компьютером. |
| `кресло с сетчатой спинкой` | Если материаловый гайд и серия `SKY` начнут конкурировать за один запрос. |
| `кресло для open space` | Если B2B-запросы дадут отдельный спрос от статьи про офисную команду. |
| `как собрать кресло Aerocool` | Если post-purchase запросы начнут расти и FAQ станет слишком длинным. |
| `где купить кресло Aerocool в Украине` | Только если есть точная коммерческая логика покупки, доставки и контактов. |

### Этап 3. Подключить GSC И Пересчитать Приоритеты

После появления Google Search Console данных:

- строки с показами и низким CTR идут в сниппетную доработку;
- строки с позицией `8-20` идут в контентное усиление;
- строки с несколькими конкурирующими URL идут в anti-cannibalization разбор;
- строки без показов после 90 дней остаются в low-priority growth или удаляются из активного плана.

## Правила Для Украинской И Русской Версий

- Не переводить ключи дословно, если в языке другой нормальный спрос.
- Структура аргумента должна совпадать, формулировка может отличаться.
- Если добавлен блок в украинскую статью, добавлять соответствующий блок в русскую версию.
- Внутренние ссылки должны вести на URL текущего языка.
- `any`-ключи использовать для моделей, механизмов и терминов, которые одинаковы в обоих языках.

## Где Использовать Новые Ключи

| Тип Ключа | Где Использовать | Где Не Использовать |
| --- | --- | --- |
| `купить`, `цена`, `наличие` | Товар, каталог, контакты | Информационные статьи без product facts |
| `как выбрать` | Evergreen-гайд | Товарный `title` |
| `что такое` | Объясняющая статья | Каталог |
| `для программиста`, `для видеозвонков` | Сценарный блок статьи | Все товары подряд |
| `гарантия`, `ремонт`, `сервис` | FAQ, контакты, post-purchase | Серии без подтвержденных условий |
| `до 10000 грн`, `до 12000 грн` | Каталог и товар при актуальной цене | Тексты без проверки `priceValidUntil` |
| `характеристики + модель` | Товарная страница конкретной модели | Статьи, если пользователь уже ищет конкретную модель |

## Контентная Формула Блока

Каждый новый блок под группу ключей должен иметь:

1. Сценарий пользователя.
2. Критерии выбора.
3. Связь с конкретными сериями или моделями.
4. Ограничение или честное “может не подойти”.
5. Внутреннюю ссылку на следующий шаг.

Пример структуры:

```text
## Кресло для видеозвонков в home office

Коротко описать сценарий.
Объяснить критерии: посадка, внешний вид, материал, шум движения.
Показать, какие серии подходят и почему.
Указать, когда лучше выбрать другой вариант.
Дать ссылку на каталог, серию или товар.
```

## Риски

| Риск | Как Предотвращать |
| --- | --- |
| Переспам | Один главный интент на страницу, ключи распределять по смыслу. |
| Каннибализация | Перед новой страницей проверять CSV и текущий `target_url`. |
| Неподтвержденные обещания | Цена, наличие, доставка, гарантия и сервис только из product facts и FAQ. |
| Слабые страницы | Новый URL создавать только после GSC/рекламного подтверждения спроса. |
| Дубли языка | `uk` и `ru` развивать синхронно, но писать естественно. |

## Практический Порядок Работы

1. Выбрать кластер из CSV.
2. Проверить текущий `target_url`.
3. Решить: усилить страницу, держать growth-строку или создавать новый URL.
4. Добавить блок по контентной формуле.
5. Добавить 2-4 внутренние ссылки.
6. Проверить, что не появились дубли с соседними статьями.
7. После публикации ждать данных GSC и пересчитывать приоритет.
