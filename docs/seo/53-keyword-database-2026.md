# База Ключевых Слов Aerocool 2026

Обновлено: 2026-06-13.

Этот документ фиксирует рабочую базу ключевых слов проекта `Aerocool Ukraine` после анализа всего публичного контента сайта и текущей SEO-документации. Полная табличная база находится рядом в [53-keyword-database-2026.csv](53-keyword-database-2026.csv), а стратегический порядок развития семантического ядра описан в [72-semantic-core-keyword-strategy-2026.md](72-semantic-core-keyword-strategy-2026.md).

Документ не обещает позиции в Google. Его задача — дать команде единый словарь для аналитики, развития контента, расширения семантики, проверки каннибализации и планирования новых страниц.

## Что Было Проанализировано

- `100` отслеживаемых markdown-файлов в `content/**/*.md`.
- `50` украинских и `50` русских отслеживаемых страниц.
- Главная, About, Contact, FAQ, Image License, Search, Contact Success.
- Хабы `/products/`, `/articles/`, `/news/`.
- `3` страницы серий в двух языках: `SKY`, `WING`, `XTAL`.
- `12` товарных вариантов в двух языках.
- `16` evergreen-статей в двух языках.
- `9` новостей в двух языках.
- `hugo.yaml`, где заданы глобальные `description` и `params.keywords`.
- [18-seo-keyword-map-2026.md](18-seo-keyword-map-2026.md), где уже описана стратегическая карта интентов.

Служебные страницы `/search/` и страницы подтверждения формы не считаются SEO-посадочными, но учитывались как часть корпуса, чтобы не принять их за новые цели для ранжирования. Страницы `/image-license/` и `/ru/image-license/` считаются поддерживающими legal/trust URL: они не являются коммерческими посадочными, но заведены в базу, потому что поддерживают ImageObject license metadata и доверительный слой сайта.

## Вывод По Текущему Покрытию

Сильнее всего проект уже покрывает:

- брендовый спрос: `aerocool`, `aerocool украина`, `aerocool україна`, `официальный сайт aerocool`;
- серии и модели: `SKY`, `SKY Light`, `SKY 360`, `WING`, `XTAL`, `Racer`, `Loft Air`, `Mesh`;
- технические сущности: `Synchronous Tilt`, `SYNC4`, `SYNC5`, `7D`, `8D`, `11D`, `4D X 360`, `3D X 360`;
- материалы и поверхности: `Leatherette`, `Loft Air`, `Mesh`, `breathable mesh`, вентиляция;
- сравнения: `SKY Light или SKY 360`, `WING или XTAL`, `Racer / Loft Air / Mesh`;
- базовые коммерческие типы: игровое, офисное, компьютерное и эргономичное кресло;
- новые сценарные кластеры: работа за компьютером, home office, долгая работа и посадка, настройка кресла после покупки, офисная команда, переговорная, команда разработчиков, сравнение всех серий.
- расширенный слой материалов: уход за `Leatherette`, уход за `Mesh`, вентиляция, сравнение `WING Mesh / WING Loft Air` и `XTAL Racer / XTAL Mesh`;
- purchase-слой без непроверенных обещаний: `купити ігрове крісло aerocool`, `купить игровое кресло aerocool`, `купити офісне крісло aerocool`, `купить офисное кресло aerocool`.

Главная зона роста теперь не в простом добавлении новых ключей, а в управлении уже опубликованной семантикой: защита от каннибализации между похожими гайдами, подключение данных Google Search Console, уточнение приоритетов по реальным показам и расширение коммерческих фактов только там, где они подтверждены командой Aerocool Украина.

В этом проходе база подготовлена для дальнейшей аналитики: у строк больше нет обобщенной цели `product pages`, хабы `/articles/` и `/news/` получили собственные keyword-строки, страницы прав на изображения получили support-ключи, добавлены новые material/B2B/purchase long-tail строки, а CSV содержит пустые поля для будущей выгрузки Google Search Console.

## Как Пользоваться Базой

CSV-файл содержит строки вида:

| Поле | Значение |
| --- | --- |
| `id` | стабильный идентификатор строки базы |
| `lang` | язык ключа: `uk`, `ru`, `any` |
| `keyword` | ключевая фраза |
| `cluster` | мастер-кластер: бренд, категория, серия, товар, материал, механизм, сравнение, поддержка |
| `subcluster` | уточнение внутри кластера |
| `intent` | поисковый интент |
| `priority` | `P0`, `P1`, `P2`, `P3` |
| `target_url` | текущая или рекомендуемая целевая URL |
| `target_scope` | тип цели: `single_url` для одной страницы или `url_group` для группы URL через вертикальную черту |
| `status` | `current`, `supporting`, `growth` |
| `funnel_stage` | этап пользовательского пути: `navigation`, `awareness`, `consideration`, `purchase`, `support` |
| `business_value` | бизнес-ценность ключа: `high`, `medium`, `low` |
| `gsc_impressions` | показы из Google Search Console после подключения данных |
| `gsc_clicks` | клики из Google Search Console после подключения данных |
| `gsc_ctr` | CTR из Google Search Console после подключения данных |
| `gsc_avg_position` | средняя позиция из Google Search Console после подключения данных |
| `country` | страна среза, если строка обновлена по данным аналитики |
| `device` | устройство среза, если строка обновлена по данным аналитики |
| `last_checked` | дата последней проверки или импорта аналитики |
| `notes` | пояснение, как использовать ключ без переспама |

Практическое правило: перед созданием новой страницы сначала фильтруйте CSV по `cluster`, `intent`, `funnel_stage`, `business_value` и `target_url`. Если ключ уже имеет сильную целевую страницу, лучше усилить эту страницу, а не создавать дубль. Если `target_scope` равен `url_group`, целевые URL внутри `target_url` разделены через ` | ` и должны проверяться как группа.

## Приоритеты

| Приоритет | Значение |
| --- | --- |
| `P0` | настоящее ядро проекта: бренд, каталог, серии, транзакционные товарные запросы и главные money-keywords |
| `P1` | опубликованные поддерживающие кластеры, важные гайды, сравнения, материалы, модели и сценарии выбора |
| `P2` | зона роста: long-tail, FAQ-блоки, новые подразделы, будущие статьи и сравнения |
| `P3` | низкочастотные или вторичные сценарии, которые не должны размывать структуру сайта |

## Контроль Каннибализации

Эти страницы близки по теме, но должны отвечать на разные поисковые задачи:

| Основная задача | Primary URL | Не Дублировать |
| --- | --- | --- |
| Общий выбор кресла Aerocool | `/articles/how-to-choose-aerocool-chair/` и `/ru/articles/how-to-choose-aerocool-chair/` | Не превращать в подробный гайд по настройке, осанке или офисной закупке. |
| Работа за компьютером полный день | `/articles/chair-for-computer-work/` и `/ru/articles/chair-for-computer-work/` | Не дублировать общий гайд выбора; держать фокус на рабочем дне, столе и посадке. |
| Home office | `/articles/best-chair-for-home-office/` и `/ru/articles/best-chair-for-home-office/` | Не конкурировать с общим рабочим сценарием; раскрывать домашнее пространство, видеозвонки, смешанное использование. |
| Осанка и долгая работа | `/articles/chair-for-posture-and-long-work/` и `/ru/articles/chair-for-posture-and-long-work/` | Не давать медицинских обещаний и не дублировать технический гайд по регулировкам. |
| Регулировки `7D/8D/11D` | `/articles/how-to-choose-chair-by-adjustability/` и `/ru/articles/how-to-choose-chair-by-adjustability/` | Держать фокус на механике регулировок, а не на общем выборе серии. |
| Настройка после покупки | `/articles/chair-setup-after-purchase/` и `/ru/articles/chair-setup-after-purchase/` | Не конкурировать с предкупочными гайдами; это post-purchase инструкция. |
| Офисная команда | `/articles/chairs-for-office-team/` и `/ru/articles/chairs-for-office-team/` | Не смешивать с индивидуальным home office; держать B2B/командный сценарий. |
| Все серии `SKY/WING/XTAL` | `/articles/sky-vs-wing-vs-xtal/` и `/ru/articles/sky-vs-wing-vs-xtal/` | Не дублировать парные сравнения `SKY Light vs SKY 360` и `WING vs XTAL`. |

## Ключевые Зоны Роста

| Зона | Что Делать |
| --- | --- |
| Безбрендовый commercial | Усиливать существующие хабы и статьи, не создавать дубли под каждую форму запроса. |
| Home office | Расширять практические критерии: длительность дня, вентиляция, подлокотники, регулировка сиденья, сценарии квартиры/офиса. |
| Осанка и эргономика | Писать аккуратно: без медицинских обещаний, через настройку посадки, поддержку поясницы, подлокотники и движение. |
| Материалы | Текущая база уже включает уход, вентиляцию и сравнение поверхностей. Следующий шаг — усилить эти блоки на существующих страницах и выделять отдельные страницы только после данных GSC. |
| Механики WING и XTAL | Усилить `dual backrest` и `fully replaceable design` видимыми объяснениями, FAQ и внутренними ссылками. |
| B2B-сценарии | Развивать `кресла для переговорной` и `кресла для команды разработчиков` как подразделы статьи про офисную команду, пока нет подтвержденного спроса на отдельные посадочные страницы. |
| Транзакционные фразы | Использовать только там, где подтверждены цена, наличие, доставка, гарантия и условия покупки. |
| Search Console | После production-индексации заполнить `gsc_*`, разделить сильные `any`-строки на `uk/ru` и пересчитать приоритеты. |

## Правила Расширения

- Не создавать новую страницу, если текущая URL уже точно закрывает интент.
- Не смешивать на одной странице `купить`, `сравнить`, `что это` и `как выбрать`, если от этого расплывается основная задача.
- Украинские и русские ключи развивать параллельно, но не дословным переводом.
- Для `лучшее`, `краще`, `лучший` обязательно раскрывать критерии выбора.
- Для характеристик вроде `BIFMA`, `price`, `availability`, `warranty`, `delivery` использовать только подтвержденные данные.
- Не использовать `meta keywords` как основную SEO-стратегию.
- Не вставлять ключевые слова в текст механически. Ключ должен быть подтвержден смыслом блока, ссылками, характеристиками или FAQ.
