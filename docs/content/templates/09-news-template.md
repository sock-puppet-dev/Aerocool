# Шаблон Новости

Обновлено: 2026-06-18.

Использовать для брендовых анонсов и обновлений в `content/news/<slug>/index.md` и `index.ru.md`.

Редакционный ориентир проекта: содержательная новость обычно имеет `5000+` знаков основного текста на каждую языковую версию. Это не SEO-требование: новость нельзя растягивать evergreen-объяснениями ради объема.

Как пользоваться новичку: новость должна описывать реальный инфоповод, а не просто имитировать статью. Создай папку новости, добавь `index.md`, `index.ru.md`, локальную обложку `01-front.webp`, затем заполни front matter и текст по структуре ниже.

## Поля Метаданных Страницы

```yaml
---
title: "Заголовок новости"
linkTitle: "" # необязательно: короткое имя для breadcrumbs
h1: "" # необязательно
description: "Краткое описание инфоповода и его значения для каталога Aerocool."
summary: "Одно предложение о том, что произошло и кому это особенно полезно."
date: 2026-04-21T10:00:00+03:00
lastmod: 2026-04-21T10:00:00+03:00
slug: "<slug>"
image: "01-front.webp"
cover:
  image: "01-front.webp"
  alt: "Конкретная модель, серия или событие в кадре"
  relative: true
  hiddenInSingle: true
schema_types: ["website", "news", "organization", "breadcrumbs"]
---
```

Сразу после front matter, если у новости есть локальная обложка в папке страницы:

```md
{{< seo-image
  src="01-front.webp"
  width="1536"
  height="1024"
  alt="Конкретное описание модели, серии или события"
  title="Aerocool <TITLE>"
  loading="eager"
  preload=true
  fetchpriority=high
  class="w-full rounded-2xl"
  sizes="(min-width: 1198px) 1150px, (max-width: 768px) calc(100vw - 28px), calc(100vw - 48px)"
/>}}
```

## Редакционные Правила

- Видимый `H1` страницы рендерит шаблонный слой; в теле новости не добавлять markdown `# H1`.
- Если нужно развести SEO `title` и визуальный заголовок, использовать необязательное поле `h1` в метаданных страницы.
- В видимом тексте новости не использовать обратные кавычки для inline-code. Точные технические обозначения, характеристики, SKU/MPN/GTIN, размеры, рейтинги и значения из таблиц выделять обычным жирным форматом: `**11D**`, `**SYNC5**`, `**75 мм**`. Широкие коммерческие SEO-фразы писать обычным текстом или обычным жирным выделением.
- Новость должна быть про реальный анонс, запуск, обновление или изменение.
- Не использовать страницу новости для постоянно актуальных сравнений.
- Не превращать новость в пустой пресс-релиз; даже при объеме `5000+` знаков материал должен объяснять контекст, сценарии и следующий шаг.
- Не копировать в новость полный evergreen guide. Новость фиксирует факт изменения и ведет в статью, где термин или механизм объяснен подробно.
- После финального вывода новости не начинать второй смысловой блок.
- Обязательно вести пользователя в соответствующую серию, модель или каталог.
- Текущий рекомендуемый стандарт для news bundle в проекте: `image + cover.image + seo-image` с локальным `01-front.webp`.
- Полный news image bundle включает `01-front.webp`, crops `01-front-16x9.webp`, `01-front-4x3.webp`, `01-front-1x1.webp` и обычно один inline-файл `02-*` **1200x800**; второй inline допустим только для запуска серии или нескольких материалов.
- Для русской версии использовать тот же локальный файл и локализованные `alt` / `title`. `jsonld` в shortcode не добавлять.
- Для первого изображения использовать полный LCP-набор из шаблона: `loading="eager"`, `preload=true`, `fetchpriority=high` и проектный `sizes` под `.main`. Shortcode сам создаст WebP `srcset` и fallback.
- Если первое изображение использует нестандартный `sizes`, добавить такой же `seo_image_sizes` во front matter.
- Если у новости нет собственного изображения в папке страницы, допустим fallback `image: "images/default-news.webp"`, но это уже запасной сценарий, а не основной стандарт.
- Новость должна усиливать не только брендовые запросы, но и широкие коммерческие кластеры, если инфоповод действительно это позволяет.
